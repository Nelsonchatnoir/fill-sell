AXE 1 — MOIS (bébé) :
  LBC bébé   : Prématuré/44 → 36 mois/98 cm (chaîne "mois / cm")
  Beebs      : Prématuré → 24 mois (chaîne "N mois (X-Y cm)")
  Vinted     : Prématuré → 24-36 mois (chaîne "N-N mois / NN cm")
  eBay bébé  : "1 mois"…"24 mois" + cm nus 50-98 (2 systèmes mélangés)
AXE 2 — ANS (enfant) :
  Vinted     : 3 ans/98 → 16 ans/176 (chaîne "N ans / NN cm")
  Beebs      : 3 → 10 ans puis 12/14/16 (chaîne "N ans (X-Y cm)")
  LBC Mode/Enfant : 3 ans/98 → 16 ans/176 + "18 ans / 182 cm +"
  eBay enfant: lettres 2XS-4XL + cm nus 62-176 + "2 ans".."16 ans" (3 sys.)
AXE 3 — POINTURES (EU nues) :
  Vinted 15→40 │ Beebs 15→41 │ eBay EU 15→40 (+ demi/tiers pointures)
  LBC : pas de champ pointure enfant structuré relevé (via desc./Mode).

OBLIGATOIRE (bloque la publi si absent) : eBay Taille + Pointure EU
(required=true). Les 3 autres : taille facultative.

FORMAT : partout (sauf pointures + eBay), la taille enfant est une CHAÎNE
COMBINÉE âge+cm → un nombre nu n'est JAMAIS égal en exact, mais est contenu
comme mot ("3" ⊂ "3 ans / 98 cm") → garde anti-fuzzy Étape D nécessaire.
eBay + pointures = nombres nus purs (56, 92, 128, 15..41) → collision
directe avec tailles adulte (34-52) et entre elles.

LEBONCOIN — l'hypothèse "pas de champ taille" du code (generate-listing
L45-51) est FAUSSE : LBC a des tailles structurées (bébé 0-36 mois +
Mode/Univers=Enfant 3-18 ans). Le vrai enjeu LBC = poser Univers=Enfant.

★★★ LE RISQUE "TAILLE ADULTE EN SILENCE" EST LARGEMENT REFUTÉ ★★★
"Mode > Vêtements" a un champ "Univers*" (OBLIGATOIRE) et LBC a AUTO-détecté
depuis le titre :
   Univers = Enfant │ (Style/genre) Garçon │ Type = Manteaux & Vestes
   Taille = "8 ans / 128 cm" │ Couleur = Bleu / Ciel
=> Quand l'Univers est un univers ENFANT, la grille Taille est une grille
   ENFANT (ans / cm), PAS S/M/L adulte.

Univers* — valeurs : Femme | Maternité | Homme | Enfant | Fille | Garçon |
   Unisexe.
Taille (Univers enfant) — format COMBINÉ "ans / cm", 15 valeurs :
   3 ans / 98 cm │ 4 ans / 104 │ 5 ans / 110 │ 6 ans / 116 │ 7 ans / 122
   8 ans / 128 │ 9 ans / 134 │ 10 ans / 140 │ 11 ans / 146 │ 12 ans / 152
   13 ans / 158 │ 14 ans / 164 │ 15 ans / 170 │ 16 ans / 176 │ 18 ans / 182 cm +

COUVERTURE LBC COMPLÈTE (2 catégories complémentaires, tailles structurées) :
   • Famille > Vêtements bébé   → 0-36 mois (Prématuré/44 → 36 mois/98)
   • Mode > Vêtements (Univers=Enfant/Fille/Garçon) → 3-18 ans (ans/cm)

⚠️ VRAI RISQUE RÉSIDUEL (ramené, mais réel) : la grille enfant n'apparaît
   QUE SI l'Univers est enfant. Ici LBC l'a déduit du mot "8 ans" dans le
   titre. Si notre titre/attributs LLM ne signalent PAS clairement l'enfant,
   l'Univers peut défaulter sur Femme/Homme → Taille adulte. Le risque n'est
   donc plus "LBC n'a pas de champ" mais "il faut que leboncoin.js POSE
   explicitement Univers=Enfant/Fille/Garçon (depuis genre) AVANT de choisir
   la Taille". C'est un point d'implémentation Étape C, pas une fatalité data.

★ POINTS CLÉS POUR L'IMPLÉMENTATION :
  • eBay est la SEULE plateforme qui expose la Taille/Pointure enfant comme
    aspect OBLIGATOIRE (required=true) → un mapping enfant sans taille valide
    = publication refusée. C'est la contrainte la plus dure des 4.
  • eBay MÉLANGE 3 systèmes (lettres + cm-nu + ans) dans un seul dropdown
    "Taille" enfant. Le référentiel Étape B doit pouvoir émettre l'un OU
    l'autre selon ce qu'eBay accepte pour la feuille.
  • ⚠️ FUZZY (Étape D) : eBay regorge de NOMBRES NUS valides — cm (56,62,
    68…176), pointures (15…40), et "Taille unique". Un "92" est une Taille
    cm bébo/enfant ET une "92 cm" de Tour de taille. Surface de collision
    fuzzy énorme → la garde doit être stricte sur les champs taille eBay.
  • Bornes : bébé eBay va à 24 mois / ~98 cm (cohérent LBC 36 mois près) ;
    enfant eBay 2→16 ans. Pointures EU 15→40 (comme Vinted).
